# MusicCat

MusicCat is a macOS‑native Apple Music archiving toolkit written in C++. It captures currently playing tracks, extracts rich metadata and artwork, records lossless audio through a virtual device, transcodes outputs, and organizes the final library into album‑based folders.

MusicCat runs as a lightweight command‑line listener. It continuously monitors Apple Music playback and automatically records tracks when playback begins, then performs post‑processing such as metadata embedding, artwork extraction, format conversion, and library organization.

---

# Features

- Detect whether Apple Music is running
- Detect whether music is currently playing
- Monitor track changes
- Read detailed Apple Music metadata, including:
  - title / artist / album
  - album artist / composer / genre / lyrics
  - year / track number / disc number
  - bpm / rating / play count / sample rate / bit rate
- Export and preserve current track artwork
- Record playback audio as **24‑bit FLAC** via macOS CoreAudio virtual device
- Convert FLAC recordings to **M4A**
- Write metadata and embedded artwork back into **FLAC and M4A**
- Organize exported files into **album‑based library folders automatically**
- Apply album cover as **macOS folder icon**

---

# Project Philosophy

MusicCat treats streaming playback not as a disposable listening session but as a source for building a **structured and durable personal music archive**.

The project focuses on:

- metadata integrity
- artwork preservation
- lossless capture
- album‑oriented library organization

The objective is not merely recording audio, but preserving a musical work together with its contextual information.

# Requirements

MusicCat runs only on **macOS** and relies on several external tools.

Install dependencies with Homebrew:

```
brew install ffmpeg sox fileicon
```

Required components:

- **Apple Music** (playback source)
- **FFmpeg / FFprobe** (transcoding, metadata processing)
- **SoX** (audio recording)
- **fileicon** (setting album cover as macOS folder icon)
- **Virtual Audio Device** (CoreAudio loopback device)

Example virtual audio drivers:

- Loopback
- BlackHole
- Soundflower
- other CoreAudio virtual devices

# Virtual Audio Setup

MusicCat records audio from a **virtual input device**.

Before using it, create a virtual audio device and route **Apple Music output** into that device.

Example configuration:

```
Apple Music → Virtual Audio Device → MusicCat (recording)
```

Typical device name example:

```
Apple Music Virtual Device
```

---

# Recommended Preparation

For best results:

1. Place the Apple Music tracks you want to capture into a **dedicated playlist**.
2. Download the tracks locally in Apple Music before playback.

Downloading tracks first helps avoid recording interruptions caused by network instability.

# CLI Usage

MusicCat is designed as a **terminal‑based CLI application**.

Basic syntax:

```
MusicCat <command> [options]
```

---

# Commands

## Start the Listener

```
MusicCat ready
```

This starts the Apple Music playback listener.

MusicCat will:

1. monitor Apple Music state
2. detect when playback begins
3. start recording automatically
4. stop recording when playback stops or the track changes
5. process the resulting audio

The process runs continuously until terminated.

---

## Set Output Directory

```
MusicCat -o <path>
```

Example:

```
MusicCat -o ~/Music/MusicCat
```

If not specified, the default output directory is typically the Desktop.

---

## Set Virtual Audio Device

```
MusicCat -vd <deviceName>
```

Example:

```
MusicCat -vd "Apple Music Virtual Device"
```

MusicCat records audio through this device using **SoX + CoreAudio**.

---

## Show Version

```
MusicCat -version
```

or

```
MusicCat -v
```

---

## Print Logs

```
MusicCat log
```

Displays runtime logs generated by the application.

---

## Help

```
MusicCat help
```

Chinese and Japanese help is also available:

```
MusicCat zh
MusicCat ja
```

# Architecture

The simplified processing pipeline of MusicCat looks like this:

```
Apple Music
     │
     ▼
Virtual Audio Device
     │
     ▼
     SoX
     │
     ▼
    FLAC Recording
     │
     ▼
   FFmpeg
(metadata + artwork embedding)
     │
     ▼
    M4A Output
     │
     ▼
 Album Library Organizer
```

MusicCat listens to Apple Music playback events, records audio through a CoreAudio virtual device using **SoX**, then post‑processes the captured FLAC using **FFmpeg** to embed metadata, attach artwork, convert formats, and organize the results into a structured album library.

# Typical Workflow

```
MusicCat -o ~/Music/MusicCat
MusicCat -vd "Apple Music Virtual Device"
MusicCat ready
```

Then simply start playing music in Apple Music.

MusicCat will automatically:

- detect playback
- start recording
- capture metadata
- export artwork
- convert FLAC → M4A
- embed metadata and cover art
- organize files into album folders

Additional behaviors:

# Internal Workflow (Simplified)

MusicCat uses an event‑driven listener model:

- detect whether Apple Music is running
- detect playback start/stop
- detect track change

Recording lifecycle:

1. read metadata from Apple Music
2. start FLAC recording via SoX
3. stop recording when playback stops or track switches
4. validate recording duration
5. attach metadata and artwork
6. convert FLAC → M4A
7. organize files into album folders

# MusicCat

MusicCat is a macOS-native Apple Music background recording CLI written in C++. It runs quietly alongside regular playback, records through a virtual audio device, preserves track metadata and artwork, and organizes processed output into album-based folders.

MusicCat is designed to reduce manual recording work. It monitors Apple Music playback state, starts and stops recording automatically, skips incomplete output caused by track switching or unstable playback, and prepares structured audio results for later listening, archiving, or dataset work.

---

# Features

- Run as a lightweight macOS CLI listener
- Detect whether Apple Music is running
- Detect whether music is currently playing
- Monitor playback state and track changes
- Read detailed Apple Music metadata, including:
  - title / artist / album
  - album artist / composer / genre / lyrics
  - year / track number / disc number

- Write metadata and embedded artwork into processed audio files
- Organize completed output into **album-based library folders automatically**

- Run with user-scoped cache, log, and support files outside the project directory
